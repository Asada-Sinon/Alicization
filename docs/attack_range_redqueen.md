# 可遗传攻击距离 vs 猎物逃逸：本项目第一个可测量的红皇后动态

这份文档记录本项目第一次**双侧协同演化（Van Valen 1973 红皇后）**的实现与实测：给
捕食者一条可遗传的「攻击距离」基因、给食草动物一条对称的「逃逸」基因，让两者在捕食
判定里直接对抗，然后用 6 配对种子 ×20000 步测它们是否互相追着往上进化。落地规格来自
`docs/trait_roadmap.md` §7.3（`attack_range` 可遗传的逐文件规格）与 §5（代价货币原则）；
捕食参数现状与不要冲突的约束来自 `docs/carnivore_riparian.md`。

标记体例沿用 `docs/biology.md`：**[现实]** 已发表事实、**[本世界实测]** 本代码库跑出来
的数字、**[对应]** 落到哪段代码、**[提案，非结论]** 尚未验证的设计建议。

---

## 0. 一句话结论

**实现的是双侧版，红皇后动态成立且干净**——逃逸性状只在它真正有用（捕食是真威胁）时
才演化起来（双侧 herb_escape 1.91 vs 单侧中性漂变 0.50，6/6 同向，p=0.031），逃逸的
上升**滞后**攻击约 3500 步（猎物对上升的捕食压力做出响应），并把捕食者辛苦投资出来的
攻击距离几乎完全抵消掉——**双侧的「有效攻击距离」被逃逸压回到 6.28，约等于 6.0 的起点，
而单侧无对手时一路涨到 8.20**。捕食者「拼命奔跑只为留在原地」。生态后果：双侧捕食者
占比稳定在约 14%，单侧涨到 25%——**猎物防御把可维持的捕食者密度砍掉约一半**（6/6 同向，
p=0.031）。

**但要诚实指出一条与路线图预测不符的地方**：`docs/trait_roadmap.md` §7.3 预测「双侧版
攻击范围**方差显著高于**单侧」——**这条被证伪**。两臂的攻击距离方差没有显著差异
（attack_std p=0.69；后半程时间方差 p=1.0），两臂的攻击距离都收敛到同一个由能量税
决定的均衡（约 8.2），并在约 15000 步后**同样地趋于平台**。红皇后在这里表现为一个
**达到协同演化均衡的瞬态**（攻击/逃逸各自爬升到平台、有效距离被压回基线、密度减半），
**不是**一个方差越拉越大的失控军备竞赛。逃逸的效应体现在**密度和有效距离**上，不体现
在攻击距离的绝对高度或方差上——原因见 §4.3。

## 1. 改了什么：给协调用的精确清单

主 agent 会在另一个 agent 改动的感知几何（`sense_grid`/`k_neighbors`/邻居系统）上把这套
结论重验一遍，所以先把**改了捕食的哪些常量与逻辑**列清楚。全部改动在 commit（实现）
`3ee22ad` 里，本文档不重复注释文字，只列会影响捕食判定的部分。

**逻辑改动（`underworld/dynamics.py`）：**

- `predation()`：原判定 `eligible = ... & (dist < cfg.attack_range) & ...`。
  **[对应]** 现在 `cfg.attack_range`（常量 6.0）替换为按攻击方个体读取的
  `attack_range_of(state.genome, cfg)`，再减去按被攻击方读取的 `escape_of` —— 即
  **有效攻击距离 = 攻击方 reach − 猎物 escape**。两条各由消融开关门控
  （`attack_range_heritable`、`prey_escape_enabled`），关掉时退回常量/无逃逸，
  是 Python 常量 baked 进 jit（仿 `peer_channel_enabled`），关掉的臂把对应分支编译掉。
- `metabolize()`：新增两个可选参数 `attack_range`、`escape`，在**能量账本**上加税——
  `attack_cost * relu(attack_range − 6.0) * diet` + `escape_cost * escape * (1−diet)`。
  **绝不碰 `thirst()` 的水账本**（`docs/trait_roadmap.md` §5 的代价货币原则：水账本会
  让性状栽进幼体渴死删失陷阱，重演体型基因的方向反转）。攻击税按 `diet` 缩放、逃逸税
  按 `1−diet` 缩放，所以中性漂变的一方（食草动物的攻击基因、食肉动物的逃逸基因）几乎
  不付税、干净漂变。仿照已验证安全的 `carn_cost` 记账方式。

**新增的捕食相关常量（`underworld/config.py`）：**

| 常量 | 值 | 含义 |
| --- | --- | --- |
| `attack_min` / `attack_span` | 1.0 / 10.0 | `attack_range_of` 映射到 [1, 11]；基因 0 → 6.0 = 旧 `attack_range` 常量，新建群第一步与旧世界一致 |
| `attack_cost` | 0.012 | 每单位超出 6.0 基线的攻击距离的能量税/步，按 `diet` 缩放 |
| `escape_span` | 12.0 | `escape_of` 映射到 [0, 6]；基因 0 → 0（无逃逸），任何逃逸都是演化出来的 |
| `escape_cost` | 0.012 | 每单位逃逸的能量税/步，按 `1−diet` 缩放 |
| `attack_mutation_sigma` / `escape_mutation_sigma` | 0.02 / 0.02 | 与 size/invest 同的慢速性状突变率 |
| `attack_range_heritable` / `prey_escape_enabled` | True / True | 消融开关；单侧臂把后者设 False |

**未改动的捕食常量**（与 `docs/carnivore_riparian.md` 协调）：`pred_rate=0.5`、
`pred_efficiency=0.7`、`meat_water_frac=0.3`、`diet_delta=0.15`、`carn_cost=0.15`
**全部原封不动**——本工作没有碰 `carnivore_riparian.md` 正在讨论的那批参数，代价改走
了新的能量税这条独立通道，不与河岸驻留主线的参数重叠。

**一条与感知几何直接相关、主 agent 必须复核的约束**：`attack_span` 特意让
`attack_max = 11.0` 严格封在当前 sense cell（`world_size/sense_grid = 512/24 = 21.3`）
以下——演化出的长 reach 若超过 sense cell，会咬到邻居表在 3×3 块里没收集到的猎物而
**静默失败**（和 `vision_radius` 同一条天花板，见 `docs/vision_radius_heritable.md`）。
`check.py --contracts` 已经把 `sense_cell >= attack_max` 加成机械不变量。**如果新几何
改了 `sense_grid`，这条守卫会自动重算**：sense cell 若变小到 11 以下，check 会红，需要
相应压低 `attack_span`；若变大则 reach 有更多演化空间（当前 8.2 均衡远未触到 11，所以
放大几何不会改变本文档的定性结论，但会松开这条天花板）。

`trait_dim` 3→5 使 `genome_size` 变、演化种群作废，golden 已用 `--bless` 重录（除 golden
外 `check.py --full` 全绿，重录后 40 项全过）。

## 2. 实验设计

- **两臂**（基因组布局相同，可直接配对）：
  - **单侧 unilateral**：`attack_range_heritable=True, prey_escape_enabled=False`。
    捕食者攻击距离可演化并计税；逃逸基因存在但无功能、不计税、纯中性漂变。
  - **双侧 bilateral**：两开关都 True。攻击与逃逸都可演化、都计税——红皇后臂。
- **6 配对种子**（seed 0–5），每种子 20000 步，chunk 500（每种子 40 个轨迹采样点）。
  默认配置（`n_max=16384`），未改任何生态常量。
- **统计纪律**（`CLAUDE.md`/`docs/conventions.md` §5）：配对 Wilcoxon（双侧 − 单侧），
  bootstrap 95% CI（10000 次重采样），报告每种子数字，**不做 Bonferroni**。n=6 配对
  Wilcoxon 的 p 地板是 0.0312（6/6 同向），本文档里凡是写 p=0.031 的都是打到这个地板。
- **未跑控制臂**（`attack_range_heritable=False`，攻击也固定）：护栏用两臂自身的
  `carnivore_frac` 判定，并对照 `docs/carnivore_riparian.md` 已测的固定基线（约 21.6%）。
  这是为省 GPU 的取舍，见 §6 局限。

**可证伪预测**（抄自 `docs/trait_roadmap.md` §7.3，执行时直接对照）：
- 单侧：攻击范围向上漂移直到代价（能量税）拉平；此后攻击方差显著低于双侧。
- 双侧：攻击范围与逃逸性状交替上升，**方差显著高于单侧**。

读三条：

1. **攻击距离在两臂里都单调爬到约 8.2–8.3**，远未触到 11 的天花板——均衡是被**能量税**
   拉平的，不是被几何封顶的（代价调对了）。
2. **逃逸只在双侧演化起来，且滞后攻击**：双侧逃逸在前约 4000 步几乎不动（约 0.4，与单侧
   的中性漂变同水平），之后随捕食者占比爬升而加速上升到 2.05。t90（到达终值 90% 的步数）
   攻击 = 14000、逃逸 = 17500，**逃逸滞后约 3500 步**——这是红皇后的相位结构：猎物防御
   是对**上升中的**捕食压力的响应，不是同步的。
3. **有效攻击距离的红皇后签名**：双侧有效距离先在约 4500 步冲到峰值 6.96，随后被追上来的
   逃逸**压回**到 6.28（≈6.0 起点）；单侧无对手，有效距离一路涨到 8.20。**捕食者付了能量
   税把 reach 投资到 8.3，猎物把其中 2.05 抵消掉，净有效距离退回起点**——Van Valen 的
   「拼命奔跑只为留在原地」在这里是可定量的。

**逃逸演化 + 捕食者占比减半**两条打到 n=6 的 p 地板、6/6 同向、CI 干净排除零——这是本
实验的两条硬结论。**攻击距离与其方差**在两臂间无显著差异——这是路线图 §7.3「方差更高」
预测的证伪，详见 §4。

### 3.3 [本世界实测] 捕食成功率随代数的变化——一条反直觉但可解释的结果

直觉预期「猎物逃逸 → 捕食成功率下降」。**实测方向相反**：双侧单步捕食成功率（食肉者本步
命中的比例）略**高**于单侧（0.520 vs 0.502，5/6 同向，p=0.062，CI 排除零）。机制是**密度
中介的补偿**：双侧捕食者占比只有单侧一半（13.7% vs 25%），种群总量反而更高（1958 vs
1577，因为食草者被吃得少），于是**每个捕食者面对的猎物池更大、竞争者更少**——找到目标
更容易，恰好抵消了逃逸带来的命中率下降。所以逃逸的效应**不是**压低单个捕食者的命中率，
而是**压低可维持的捕食者数量**。`docs/carnivore_riparian.md` §2.2 引用的 Miller et al.
(2014)「窄范围低杀伤」在这里的对应是：军备竞赛的净效应表达在**密度**这条轴上，不在
单次遭遇的成功率上。这条提醒了 `docs/carnivore_riparian.md` 已强调的一点——**不能只看
`death_predation_frac`/`hunt_success` 判断防御机制有没有生效**（Creel & Christianson 2008）。

## 4. 诚实评估：这是红皇后还是单边最优化？路线图哪条对、哪条错

### 4.1 是双侧红皇后，不是单边最优化

区分二者的直接判据（`docs/trait_roadmap.md` §7.3）是：逃逸性状是否随捕食压力协同演化，
还是各自收敛到静态值。**证据全部指向双侧**：

- 逃逸只在功能开启时演化（双侧 1.91 vs 单侧中性 0.50，p=0.031）——不是漂变。
- 逃逸滞后攻击约 3500 步——相位结构，是对上升压力的响应。
- 逃逸把有效攻击距离动态压回基线（6.96→6.28）——两性状交互，不是各自独立收敛。
- 捕食者密度被砍半——防御有真实生态后果。

### 4.2 路线图 §7.3 对了一半

- **对的一半**：双侧确实产生了协同上升，逃逸确实演化起来，确实形成本项目第一次可测量的
  红皇后（有效距离被压回、密度减半）。
- **错的一半**：§7.3 预测「双侧攻击范围**方差显著高于**单侧」——**证伪**。attack_std
  p=0.69、后半程时间方差 p=1.0，两臂方差无差异，且都在约 15000 步后趋于平台。

### 4.3 为什么攻击距离不被逃逸逼得更高、方差也不更大

这是本实验最有意思的机制发现，值得写清楚（**[本世界实测]** + **[对应]**）：

**攻击距离的均衡是被能量税、而非被猎物逃逸决定的。** `attack_cost` 是线性税，攻击距离的
边际收益（更大攻击圆 ∝ reach²，但一阶近似下命中概率的边际增量递减）随 reach 上升而递减。
当逃逸削弱了每次捕食的回报，捕食者**并不会**因此把 reach 推得更高——因为在均衡点附近，
再加 reach 的边际能量代价已经追平边际收益，逃逸只是把这个已经拉平的收益整体下移一点，
不改变「再投资 reach 不划算」这个一阶条件。系统于是通过**减少捕食者数量**（密度下降）
来卸压，而不是通过**继续升高 reach**。所以：

- 攻击距离在两臂里都停在约 8.2（税决定的均衡），双侧只比单侧高约 0.13（不显著）。
- 逃逸的全部效应转移到了**有效距离**（压回基线）和**密度**（减半）两条轴上。
- 方差不升，是因为两性状都到达了各自代价决定的平台，不再有推动方差扩大的失控反馈。

**[提案，非结论]** 若要复现路线图设想的「方差越拉越大的失控军备竞赛」，需要让攻击的
代价**次线性**或收益**不递减**（例如 reach 收益不封顶、或攻击/逃逸互为超线性放大），
使系统不存在内点均衡。当前线性能量税结构天然产向一个稳定协同均衡——这本身是一个更
接近真实生态（伏击型捕食者-猎物系统多为稳定共存而非无限升级）的结果，不是缺陷。

## 5. [本世界实测] 护栏：捕食者占比健康，无灭绝无爆炸

| 臂 | 各种子 carn%（后 1/4 均值） | 均值 | 种群均值 |
| --- | --- | ---: | ---: |
| 单侧 | 27.9 25.1 24.0 23.0 24.4 25.3 | 25.0% | 1577 |
| 双侧 | 13.8 12.8 12.2 15.3 14.1 14.1 | 13.7% | 1958 |

两臂 6/6 种子都在健康区间（约 12–28%），无一灭绝、无一爆炸。对照 `docs/carnivore_riparian.md`
测的固定 `attack_range=6.0` 基线约 21.6%：单侧把有效攻击距离升到 8.2 → 捕食更强 → 占比升到
25%；双侧被逃逸压回 → 占比降到 13.7%。**两臂正好把基线夹在中间**，符合「攻击变强则捕食者
多、猎物反击则捕食者少」的因果方向，护栏通过。

## 6. 局限与协调注记

- **伪重复（pseudoreplication，`CLAUDE.md`/`docs/conventions.md` §6）**：`terrain.build(cfg)`
  不用 RNG，6 个种子跑在**同一张地图**上，只变了建群。所以本文档所有结论严格说只对**这一
  套河系**成立。要推广到「河系一般」需交叉 `ridge_wavenumber`/`ridge_amplitude`/河源的地形
  种子——本轮未做，列为已知局限。红皇后的定性结构（逃逸滞后演化、有效距离压回、密度减半）
  在 6 个建群种子里高度一致，但这不能替代地形重复。
- **未跑控制臂**（攻击也固定的 baseline）：护栏借用了两臂自身 + `carnivore_riparian.md`
  的既有基线。若要一个同代码、同会话的干净三臂对照，补 `attack_range_heritable=False`
  一臂即可（基因组布局兼容，无需再作废种群）。
- **与并行 agent 的几何改动**：另一个 agent 在改 `sense_grid`/`k_neighbors`/邻居系统，那套
  系统也喂捕食判定。本工作**只在当前 main 的几何上实现和测量**，没碰这些常量。主 agent
  需在新几何上复验时注意 §1 末尾那条 `attack_max ≤ sense_cell` 天花板：当前 8.2 均衡远低于
  11 上限，放大几何不改定性结论；缩小几何到 sense cell < 11 会触发 check 红灯，需压低
  `attack_span`。捕食参数（`pred_rate`/`pred_efficiency`/`meat_water_frac`）本工作未动，与
  `carnivore_riparian.md` 无合并冲突。
- **认识论边界**（同 `docs/carnivore_riparian.md` §2.3 守住的那条）：我们**实现**了逃逸这个
  机制，所以观察到它演化起来只证明「我们成功构造了这个协同演化」，不能倒推为「这证明了
  自然界的红皇后」——机制是写进代码里的，不是独立涌现被发现的。

## 7. 死胡同与后续

- **[提案，非结论]** 让代价次线性 / 收益不递减以产生失控军备竞赛（§4.3）——当前线性税
  天然收敛到稳定均衡，若刻意要「方差发散」的红皇后需改代价结构，但那离真实伏击生态更远。
- **[提案，非结论]** 把逃逸做成**行为**而非静态判定折扣（例如被追时的有效速度加成），可能
  与速度性状（`trait_roadmap.md` §7.5）耦合，产生更丰富的追逐动力学——需先解决速度性状
  的「按 thrust 计费而非按 max_speed 计费」白嫖问题。
- **[提案，非结论]** 攻击/逃逸的 G 矩阵：这两个基因与 diet/size/invest 的遗传相关是否形成
  综合征？可仿 `docs/trait_stage0_analysis.md` 的零代码 G 矩阵扩展直接算。
- **地形重复**补上，把结论从「这套河系」升级到「河系一般」。
