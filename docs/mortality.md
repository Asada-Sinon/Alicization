# 死亡与选择：死因分解

这份文档记录本世界的**死亡结构**——谁死、为什么死、死在什么年龄——以及这个结构
对"某性状会不会被选择"这类主张施加的硬约束。

它单独成册的理由：死因分布是其他每一份实验文档的**分母**。`docs/experiments.md`
里的每一个负结果、`docs/trait_evolution.md` 里的每一条性状提案，都要先过这里的
竞争性风险检验才有资格被讨论。

标记约定见 `docs/biology.md` 的阅读约定一节。

## 1. 死因分解：任何"某性状会被选择"的主张都要先过这一关

### 1.1 为什么需要它

`docs/biology.md` 亲代照料一节末尾那类提案有一个共同的隐含前提：**存在一个足够强的选择压力，让被提议的性状值得付出代价。** 这个前提在本项目里被反复默认，却从未被测量过。一个降低被捕食风险的性状，如果捕食根本没在杀人，那它的适应度收益是零；一个"幼体庇护所"机制，如果幼体压根活不到被捕食的年龄，那它保护的是一个空集。

所以 `reproduction.Deaths` 把每步死亡按原因拆开并累加（`f62cc93`）。**捕食是反事实归因的**：一次死亡算作捕食，当且仅当"本步受到的伤害"正是把池子推到零以下的那一击——即没有这一击它仍为正。这刻意排除了"反复被咬放血、最后由代谢补刀"的慢性情形，那类落进 `starvation`。因此**捕食份额是下界**，不是点估计。

### 1.2 [本世界实测] 六种子 × 20000 步，约 130 万次死亡

| 死因 | 占比 | 死亡时平均年龄 |
| --- | --- | --- |
| 渴死 thirst | 83.32 ± 0.66% | 52.5 步 |
| 被捕食 predation | 9.95 ± 0.58% | 170.7 步 |
| 饿死 starvation | 6.65 ± 0.36% | 353.0 步 |
| 老死 senescence | 0.10 ± 0.00% | 3001（= `max_age`） |

平均寿命约 92 步。**渴死杀的是幼体（0.57× 平均寿命），捕食杀的是成体（1.85×）。** 捕食的真实份额被夹在 9.95%（反事实下界）与 16.6%（上界，因为饿死只有 6.65%，慢性失血最多只能占到这么多）之间。

**这直接证伪了一个已经写好的设计。** 我们原本要加一个可演化的 `size` 基因，动机是口裂限制（gape limitation）造成的体型庇护所：幼体小、会被吃，长大就逃出捕食窗口。上表说明这个故事在本世界不成立——**幼体活不到被吃的年龄**。前提没了，机制就不该加。

### 1.3 幼体缺水瓶颈，量化

这是本世界最大的单一死亡汇，且各项数字彼此吻合：

- 新生水量 = `invest_frac × parent_water` ≈ 0.356 × 5.98 = **2.13**
- 净消耗 = 0.0525 − 0.144 × `forage_water_frac` = **0.038/步** → 理论寿命 **56 步**
- 实测渴死平均年龄 **52.5 步**（吻合）
- 食草动物平均离水距离 **34.7**，实际速度 1.04 单位/步 → 需要 **~34 步的完美定向行进**
- 水感知半径 = `vision_radius + food_sample_dist` = **30 < 34.7**，且记忆槽出生即为空

**有腿、有时间、没有方向。** 这是 `docs/biology.md` 中记忆不遗传与亲代照料两节的经验依据合流的地方：跨代知识传递的正路是社会学习，而社会学习被亲代照料一节末尾那条阻塞项（同食性同类互相不可见）卡住。

### 1.4 竞争性风险（competing risks）对整个项目的含义

83% 的死亡集中在一个原因、一个年龄段，这在流行病学上叫**删失（censoring）**：其他所有选择压力都被这一条盖住了。任何针对捕食、觅食效率、投资策略的性状，其适应度差异只能在活过前 50 步的那 37% 个体身上表达，样本量和选择差都被砍掉一大截。

**这不是一个待修的 bug，而是一个必须先决定的方向问题**：要么解除瓶颈（亲代照料 / 社会学习 / 更远的水感知），让其他压力显形；要么承认本世界就是一个"幼体缺水"驱动的世界，此后所有性状演化的解释都必须先过这一条。
