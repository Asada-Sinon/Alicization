# 拆掉感知格天花板：把水感知半径解耦并做成可遗传，实测它是否缓解渴死

这份文档接续 `docs/vision_radius_heritable.md` 的结构性负结果。那份文档的结论是：
在当前世界里把 `vision_radius` 做成可遗传基因是近中性的，**根因不是代价配置，而是
`CLAUDE.md` 记录的感知格硬约束**——感知格边长把任何个体的实际视野封死在 21.33 世界
单位，仅比中性起点 21.0 高 0.33，而食草动物到水的平均缺口是 45.5 单位。那份文档在
§9 留下三条"救活它"的互斥方向，并把它们列为独立的前置工作。

**本文档执行其中第 2 条（把水感知通道从邻居块天花板解耦），分两步：**先只改几何、
不加基因，6 配对种子确认生态不崩；几何稳了再加可遗传基因，6 配对种子 True/False 对照。

**核心结论（两条，一条比 `vision_radius_heritable.md` 更强）：**

1. `[本世界实测]` **几何天花板确实被拆掉了，而且拆得干净**——水通道本就是一次地形
   场采样、从不读邻居块，所以给它一个独立的、不受感知格限制的半径，既不动
   `sense_grid`/`k_neighbors`，也不改捕食的候选邻居集。第一步把每个个体的水感知
   距离从 30 固定放大到 50（越过 45.5 缺口）后，**生态稳定**：`carnivore_frac`
   0.221 vs 0.223、`population` 2023 vs 2004、`min_pop` 1824——护栏全部守住。
2. `[本世界实测]` **但把水感知放大到 50 之后，渴死占比纹丝不动**（0.534 vs 0.549，
   p=0.438），食草动物到水距离也不动（45.6 vs 45.4）。**这比旧负结果更强**：旧结论
   是"天花板挡住了性状，测不了"；现在天花板拆了，**免费**给每个个体越过缺口的水感知
   距离，渴死依然是第一死因。**渴死瓶颈不是"看不见水"造成的。** 于是把它做成可遗传
   基因后，选择找不到向上的适应度梯度，基因不升反**降**到中性以下（True 17.2 vs
   False 20.7）——不是因为几何没抬够、也不是因为能量税太重（个体正落在税为零的
   ≤21 区间里），而是因为这条性状**没有可兑现的收益**。

标记约定同 `docs/biology.md`：`[现实]` 已确立的真实世界科学；`[本世界实测]` 在这个
模拟里跑出来的数字；`[对应]` 原则落到了哪个参数/机制；`[提案，非结论]` 尚未验证的构想。

## 1. 为什么选"解耦水通道"而不是"放大感知格"

`vision_radius_heritable.md` §9 给了两条几何路：

- **(a) 放大 `sense_grid` 格宽 + 提高 `k_neighbors`**：减小 `sense_grid` 让格子变大，
  视野天花板随之抬高。但代价链条很长：更大的格子每格装更多个体，而水边聚集极密，
  某个临水格的实际占用会远超平均；一旦超过 `k_neighbors` 容量，溢出的邻居被静默丢弃、
  对**视觉和捕食都不可见**。要压住这个 bug 就得大幅提高 `k_neighbors`，抬升
  `[n_max, 9K, …]` 邻居张量的显存与算力。**更关键的是**：`spatial.py` 的邻居索引
  "一个索引同时喂视觉和捕食"，动 `sense_grid`/`k_neighbors` 会连带改变捕食能看见的
  候选集——正是任务和 `CLAUDE.md` 反复警告的耦合。它买到的是**邻居视野**的头顶空间，
  而邻居视野（看见更远的猎物/捕食者）根本不是渴死瓶颈的杠杆。

- **(b) 把水通道从邻居块解耦**：`[对应]` 读 `sensors.sense` 第 144 行就会发现，
  **水通道本来就不读邻居块**——它是对 `terrain.water_dist` 的一次场采样，在每个
  扇区方向朝前 `food_sample_dist`(9.0) 处取一个点，`water_ch = clip(1 − wd/R)`。
  唯一把它和感知格天花板绑在一起的，是它借用了 `vision_radius` 当归一化半径 `R`。
  给水通道一个**独立**的半径 `water_vision_radius`，它的探测距离就能超过感知格，而
  邻居块（prey/pred/peer 的 `closeness`、以及捕食的候选集）**一个字节都不用动**。

**判决：选 (b)。** 三条理由：
1. **零捕食几何风险**——不碰 `sense_grid`/`k_neighbors`/邻居系统，捕食判定与之前
   逐位相同。这也让并行的 attack_range 实验可以在**未改动的邻居几何**上继续验证。
2. **最对症**——`vision_radius_heritable.md` §9 自己就说这是三条里"最对症的"，因为
   水感知本就是场采样、不受 3×3 块限制。
3. **头顶空间真实**——水感知距离 = `water_vision_radius + food_sample_dist`。把
   `water_vision_radius` 从 21 抬到 41，水感知距离从 30 变 50，越过 45.5 缺口；这是
   旧天花板(0.33 增量)的~60 倍头顶空间，而且**不需要任何裁剪**（没有邻居块要保护）。

代价：它不给**邻居视野**(prey/pred/peer)更多距离。但邻居视野不是渴死杠杆——渴死是
"看不看得见水"的问题。所以这条性状被明确地**限定在水感知通道**，这既是对
`mortality.md` §1.4"更远的水感知"路线的忠实实现，也避免了 (a) 的耦合与
`vision_radius_heritable.md` 已证伪的"邻居视野封在 0.33"的无用性。

- `underworld/config.py` `[对应]`：新增 `water_vision_radius: float = 21.0`。默认
  21.0 == `vision_radius`，是**真 no-op**（smoke golden 逐位不变，已实测）。
- `underworld/sensors.py` `[对应]`：水通道归一化半径 `cfg.vision_radius` →
  `cfg.water_vision_radius`。
- `underworld/metrics.py` `[对应]`：`inland_frac` 的 `sensor_reach` 用
  `water_vision_radius + food_sample_dist`（水传感器真正的边界）。

**第二步（加可遗传基因，改 `trait_dim` → 一次种群作废）：**
- `underworld/config.py` `[对应]`：`trait_dim` 3→4；新增 `water_vision_min=1.0`、
  `water_vision_span=40.0`（gene=0 → sigmoid 0.5 → `1+0.5·40 = 21.0` == 旧
  `vision_radius`，新种群从旧行为起步；上界 41.0 == 第一步验证过安全的天花板）；
  `vision_cost=0.001`（能量税系数）；`vision_mutation_sigma=0.02`；消融开关
  `vision_heritable: bool = True`；派生属性 `water_vision_index = brain_params + 3`。

  water_vision_min + water_vision_span·sigmoid(gene)`——**刻意不裁剪到感知格**，因为
  水通道是场采样、不受 3×3 块限制。这条"不裁剪"就是天花板被真正拆掉的地方。
- `underworld/sensors.py` `[对应]`：`vision_heritable=True` 时水通道按行读
  `water_vision_of(state.genome, cfg)`；`False` 时读固定 `cfg.water_vision_radius`。
- `underworld/dynamics.py` `metabolize` `[对应]`：加能量税
  `vision_cost·max(reach − 21, 0)`，**记在能量账本，绝不碰 `thirst` 的水账本**
  （`trait_roadmap.md` §5"解药变毒药"）。与 `vision_heritable` 同门：False 臂税为零。
- `underworld/step.py` `[对应]`：每步从基因组重算 `vision` 传给 `metabolize`。
- `underworld/genome.py` `mutate` `[对应]`：`water_vision_index` 列以
  `vision_mutation_sigma` 突变。**`crossover` 刻意不豁免**（同 invest，防止与 diet
  基因搭连、让食肉/食草视野靠搭便车而非选择分化）。
- `underworld/metrics.py` `[对应]`：append 四个字段——`mean_water_vision`、
  `water_vision_std`、`carn_water_vision`、`herb_water_vision`，全部报告**实际水感知
  半径**（`water_vision_of`）。append 不插入；`server/protocol.py` 用 `.get()` 按名
  取字段，wire 无需改。
- `tests/test_kernel.py` `[对应]`：新增 3 项——基因有界/不裁剪/可重组、能量税命中能

  中性 `vision` 参数。全 53 项通过。

**没有改的（给 attack_range 实验的明确保证）：** `sense_grid`(24)、`k_neighbors`(24)、
`vision_radius`(21)、`spatial.py` 的邻居索引、`predation` 的候选集与 `attack_range`

仍成立且含义未变。唯一改变形状的是 `genome_size` 1381→1382（`trait_dim` +1）。

## 3. 方法

`[对应]` 六配对种子（seed 0–5），每种子 20000 步、chunk 1000
（`run_headless.py 20000 1000 --seed S --json --set …`）。GPU 共享，每波 6 进程并行。
统计：配对 Wilcoxon signed-rank + 10000 次 bootstrap 的配对差 95% 区间，报告每个种子的
数字，**不做 Bonferroni**，把算过的每个 p 都报出来（`CLAUDE.md`/`conventions.md` §5）。

**功效诚实说明**：种子只变 founder、不变地形，所有空间结论只推广到*这一套河系*
（伪重复）。六种子对 `inland_frac`/`herb_water_dist` 量级的小空间位移功效不足；视野
基因本身的均值差是个体级量、方差小，六种子足以给出可信区间。golden 因 `trait_dim`
变更必然失配（种群作废），已 `--bless` 重录并在 commit 说明——这是预期，不是把 band
放宽掩盖失败。

两步的四个臂（均 trait_dim=4、genome_size=1382、同一 founder 布局）：

| 臂 | 命令 | 水感知半径 R | 能量税 |
| --- | --- | --- | --- |
| **base**（第一步基线/第二步 False） | `--set vision_heritable=False`（默认 wvr=21） | 固定 21（reach 30） | 无 |
| **wide**（第一步放大） | `--set vision_heritable=False water_vision_radius=41` | 固定 41（reach 50） | 无 |
| **True**（第二步功能臂，默认配置） | `--set vision_heritable=True` | 每个体基因 [1,41]，中性 21 | `0.001·max(R−21,0)` |
| **False**（第二步对照，== base） | `--set vision_heritable=False` | 固定 21（reach 30） | 无 |

**基线连续性核对** `[本世界实测]`：`base`(第一步 trait_dim=3 试点) thirst 0.553、
carn 0.217、pop 2024；`False`(第二步 trait_dim=4) thirst 0.549、carn 0.223、pop 2004
——两种基因布局下 False 臂逐项吻合，说明对照臂确实复现了当前 main 的世界，两步实测
对着同一个正确基线做的。

`[本世界实测]` 六配对种子，`wide`(reach 50) − `base`(reach 30)，均 vision_heritable=False
（gene 存在但不被读，纯 no-op 的中性列），trait_dim=4 同布局：

| **late_carn** | 0.218 | 0.215 | +0.004 | 0.844 | 护栏守住 |
| **population** | 2023 | 2004 | +18 | 0.625 | 不失控 |
| **min_pop** | 1824 | 1809 | +15 | 0.844 | 无接近灭绝 |

| mean_age | 352 | 371 | −19 | 0.094 | 略降，非崩 |
| inland_frac | 0.346 | 0.524 | −0.179 | 0.031 | **定义假象**，见下 |

**结论 A（几何改动安全）：** `carnivore_frac`、`population`、`min_pop`、`late_carn`
四条护栏全部守住（p 全 ≥0.6，无一接近灭绝）。**把每个个体的水感知距离从 30 免费
放大到 50 没有把生态搞崩。** 可以安全进入第二步。

**结论 B（但渴死不动——第一个强负信号）：** `death_thirst_frac` 0.534 vs 0.549
（d=−0.015, p=0.438），`herb_water_dist` 45.6 vs 45.4（p=0.844）。**给全体免费越过
45.5 缺口的水感知，渴死占比没有下降、种群也没有向水靠拢。** 这在第一步就先验地
预示了第二步：如果免费的、越过缺口的水感知都不缓解渴死，那么带能量税的可遗传版本
更不会被选择向上。（trait_dim=3 的试点波给出一致数字：thirst 0.556 vs 0.553，
p=0.844——两种布局下同一结论。）

**`inland_frac` 的 −0.179 是定义假象，不是空间位移**：`inland_frac` 数的是"离水
超过 `sensor_reach = water_vision_radius + food_sample_dist` 的个体比例"。把
`water_vision_radius` 从 21 抬到 41，阈值从 30 变 50，机械地把更多个体划进"够得着"。
**尺子动了，个体没动**——`herb_water_dist`(与阈值无关的真实距离)不变(45.6 vs 45.4)
就是铁证。所以 `inland_frac` 的大幅下降不能读作种群向内陆/向水的真实迁移。

## 6. 解读：这次天花板拆了，为什么基因还是不向上

**(a) 视野没有向上演化，功能臂反而漂到中性以下。** 路线图 §7.2 的可证伪预测是"基因
从中性 21.0 向上漂到一个中间平衡"。实测**证伪**：True 臂实际半径均值 17.20、False
20.68，True **低于**中性 21，也低于对照，方向向下（6 种子里 5/6 T<F）。CI [−6.38,
−0.88] 不含零但 Wilcoxon p=0.094 未过线；`carn_water_vision` d=−3.06(p=0.031) 稍强。

**(b) 这次的下漂是真实的负选择，不是裁剪假象。** 这是与 `vision_radius_heritable.md`
的关键区别。旧文档里 False 臂坐在 20.16，是因为 `vision_radius_of` 把表型硬裁剪在
21.33、上侧被削平造成的**截断假象**。本设计的 `water_vision_of` 是纯 sigmoid 映射到
[1,41]、**没有任何裁剪**，所以 False 臂(纯中性漂变)稳稳坐在中性 21 附近(20.68)——
证明解耦干净、没有截断。于是 True 臂 17.20 明显低于中性，是**能量税真的在起作用**：
个体没有理由为一个无收益的性状付任何能量税，于是被选择推回到税为零的 ≤21 区间，并在
该区间内漂变。**下漂是负选择+漂变的真实结果，不是数值假象。**

**(c) 瓶颈完全没被撼动。** `death_thirst_frac` d=−0.015(p=0.219, CI 含零)，渴死平均
年龄 d=+0.4 步(p=0.844)，`herb_water_dist` d=+0.40(p=0.438)。而且注意方向：True 臂
水感知**更低**(17 vs 21)却渴死略低——与"看得远→渴死少"完全反向，说明这点差异是噪声
而非机制。三条本该联动的指标**全部纹丝不动**。

**(d) 诊断——为什么不是几何、不是税，而是没有收益。** 任务要求：若基因不动，要诊断
清楚是"能量税太重"还是"几何没抬够"。两者都排除：
- **不是几何没抬够**：天花板已从 21.33 抬到 41（不裁剪），头顶空间是旧版的~60 倍，
  越过了 45.5 缺口。**第一步直接证明了这一点**——给全体**免费**的 reach-50 水感知，
  渴死 0.534 vs 0.549，不动。几何完全够。
- **不是税太重**：True 臂均值 17.20，**落在 ≤21 的税为零区间里**（税 =
  `0.001·max(R−21,0)`，R≤21 时税=0）。个体根本没在付税——它们主动待在免费区。若税
  太重，会看到基因被压到下界 1 附近，而不是停在 17-20。税只在 R>21 时才有齿，作用是
  把偶尔漂到中性以上的个体拉回，仅此而已。
- **真正的原因**：**水感知距离不是渴死瓶颈的约束变量。** 第一步已先验证明：把每个
  个体越过缺口的水感知距离免费翻倍，渴死不降。既然高水感知买不到更低的渴死风险，
  选择就没有向上的适应度梯度可用；一个无收益而有(哪怕微小)成本的性状，被选择推向
  税为零的中性以下并在那里漂变。这正是 `trait_evolution.md` §11 交割期检验的对偶：
  不是"收益被幼体死亡删失"，而是"收益根本不存在"。

**(e) 没有副作用，护栏守住。** `carnivore_frac`(0.230 vs 0.223, p=0.312)、
`population`(p=0.844)、`min_pop`(1815 vs 1809, 无接近灭绝)两臂无实质差异；`late_carn`
d=+0.011(p=0.094)、`carn_water_vision` d=−3.06(p=0.031) 等 CI 勉强不含零但按"报告
每个 p、功效不足记为弱信号"的纪律，一律不写成结论。

## 8. 结论与对路线图的反馈

**这是一个比 `vision_radius_heritable.md` 更强、更干净的负结果。** 那份文档说"感知格
天花板挡住了性状，救活它要先改几何"。本文档**改了几何**（用最对症、零捕食风险的解耦
方案，把水感知天花板从 21.33 抬到 41、不裁剪），然后发现：

- **几何天花板确实可以干净地拆掉**——解耦水通道既不动邻居系统、不动捕食候选集，也不
  触发静默不可见 bug，生态稳定（第一步护栏全过）。这条工程结论本身是正向的、可复用的。
- **但拆掉天花板并不能让这条性状起作用**，因为**渴死瓶颈的病因不是水感知距离**。免费
  给全体越过缺口的水感知，渴死不降(第一步)；做成可遗传基因，选择因无收益而把它推向
  中性以下(第二步)。**"有腿、有时间、没有方向"这个诊断里的"没有方向"被证否了**——
  现在方向(水感知)既充足又便宜，渴死照旧。

对路线图的直接反馈 `[对应]`：
- **§1.4"更远的水感知能填水缺口"被实测证伪，不止是被天花板挡住。** 天花板拆了、缺口
  被越过了，渴死仍不降。`vision_radius`/水感知这条路线应从 `mortality.md` §1.4 的三条
  候选里**划掉**，而不是"等几何改好再试"。
- **Stage 2 分叉判据**（§11：`death_thirst_frac` 是否实质下降）在这条性状上取**"未
  下降"分支**——不解锁 B 类性状，转向 `mortality.md` 另两条不依赖水感知距离的路线。
- **渴死瓶颈的真实病因需要另找。** 既然水感知距离不是约束，候选转向：往返水源的
  **水耗成本**(去程+回程，而非"知不知道水在哪")、新生儿**空记忆**的先天劣势(出生只
  给 21% 满箱 + 空记忆槽 + 平均只活~92 步)、或觅食把种群按食物拉向内陆造成的
  **水—食物空间权衡**。这些都与"看见水的距离"正交，是下一步该测的方向。

**本次改动的去留：**
- **第一步的水通道解耦（`water_vision_radius`，默认 no-op）建议保留在代码里**——它是
  一次干净的、零风险的几何解耦，默认逐位等价于旧行为，将来任何"水感知距离"相关实验
  都能直接用它，不必重写；且它证明了解耦本身安全可行。
- **第二步的可遗传基因（`trait_dim` 4）是一次种群作废，落"无收益"类负结果。** 与
  `mutation_sigma_heritable`/`vision_radius_heritable` 同类——机制正确落地、可复现、
  消融开关俱全，但性状本身近中性且不缓解瓶颈。**是否合入 main 由主 agent 决定**：若
  遵循前两个负结果性状"只留文档不留代码、不为 null 机制作废一次种群"的先例，则第二步
  的 `trait_dim` 变更不进 main，只留本文档与 `vision_heritable=False` 的对照基础设施
  描述；第一步的解耦(no-op)可独立保留。本 worktree 分支两步都已落地并通过全套检查，
