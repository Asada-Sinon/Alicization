## 优先级队列

### 1. 同类可见性——已验证，微弱正信号，但缺关键一环

**问题**：`sensors.py` 的 `prey_val`/`pred_val` 由食性差算出，两个 `diet` 相近的
个体之间两个通道**精确为零**——同食性同类彼此完全不可见，任何社会行为在架构上
无从演化。

**已做**：新增 `peer` 通道（食性*相似度*），追加在 memory 之后，`in_dim` 59→67。
另有 `peer_channel_enabled` 消融开关（`786cdb0`），置零但**保持 `in_dim` 不变**，
所以消融臂与完整臂基因组兼容、可直接比较。

**实测（3 臂 × 6 配对种子 × 20000 步，§12.2）**：`mean_age` **+29 步**
（CI [+19.0,+38.8]，p=0.031），渴死占比略降。种群和空间指标全部没动。

**下一步——补上缺的那一环**：最有说服力的指标**没有被计算出来**——"幼体到最近
成体的平均距离 vs 随机置换基线"。目前只能说"打开通道后活得久一点"，**不能说
"幼体学会了跟着成体走"**。补上这个直接证据是这一项的下一步，成本很低（只需在
run 结束时对活体状态算一次，不用重跑）。

### 2. 拆反混合装置——已跑，实验够不到问题，需要重新设计（见 §11.2）

**开关已实现并合并**（`7f1119d`）：`diet_bimodal_init`、`diet_crossover_exempt`、
`diet_mutation_asymmetric`、`assortative_mating`，默认全 True 即现有行为，可用
`--set` 单独关闭。**不改变任何张量形状，不作废种群。**

**实验结果：stage1（关掉前三层、保留同型交配）六个种子全部灭绝**，417–987 步，
99% 饿死，Fisher p=0.0022。双峰性无从测量——没有活体。

**根因已隔离**（单独只关 `diet_bimodal_init` 即复现）：崩溃与遗传三层无关，是
**founder 世代本身不可存活**。两个与那四层无关的生态公式让中间型致命：
- 采食效率 `(1-diet)^6`，在 diet=0.5 处只有纯食草者的 **1.56%**
- 捕食需要 `d_i - d_j > diet_delta = 0.15`，而单峰种群 diet 全距只有 **±0.03**，
  没有任何一对个体满足阈值

既吃不了草也吃不了肉，全部饿死。

**因此 §10.1 的"四层"表述要改成六层**：四层遗传机制"防止合并"，两层生态机制
"合并了也活不下去"，后者才是让中间型致命的那一半。

**这个实验要真正问出它想问的问题，前提是一次生态学重调**：放平 `(1-diet)^6` 的
指数或降低 `diet_delta`，让 diet≈0.5 成为"活得下去但不占优"而不是死刑。按
`CLAUDE.md` 的规矩不能在单次运行上调，且会改变已经测过的一切基线——**这是一个
独立决策，不要顺手改掉。**

### 3. 生态位构建——Stage 0 已跑，机制符号错了，修正版待做（见 §11.1）

**Stage 0 已实现并合并**（`af77898`，`trample_impact` 默认 0.0 即关闭）。

**实验结果：六种子配对，空间指标全无改善，且方向是反的**——唯一 CI 干净排除零的
是 `water_bound_frac` **上升** +0.0093（更贴近水，更糟）。

**最有诊断价值的读数**：打开反馈后踩踏场自己的 Moran's I **下降** 0.028
（CI [−0.048,−0.010]），即变得**更不聚集**。这说明**机制的符号错了**——侵蚀
承载力是**负反馈**（聚集→食物少→离开→踩踏衰减），是驱散机制，不是路径形成。

**修正版设计（下一步该做的）**：真实兽径是**正反馈**——踩踏压实地面、清除植被，
使**通行更省力**，于是更多动物走同一条线；白蚁巢、河狸坝同理，改造让该地点对后续
个体**更有吸引力**。所以应该让踩踏**降低移动成本**（`dynamics.act` 的
`forest_slow` 或 `climb_cost` 一侧），而不是降低食物承载力。**能原样复用已合并的
踩踏场机器**，只需改 `step.py` 里 `effective_capacity` 那一处的作用对象。仍是零
基因组代价，不作废种群。

**顺带确认**：`corr_trample_deficit` 在机制**完全关闭**的基线臂就已经是 0.393，
说明 piosphere signature 只是采食本身，早就存在（与 §1 一致）——这也是为什么
打开反馈几乎测不出增量。

- **Stage 3（真正主动沉积）**：`out_dim` 加一维（"投入生态位构建 vs 继续移动"），
  **作废种群**（`brain_params` 变），应与其他作废种群的改动打包做。做之前先把
  Stage 0 的符号问题解决——主动版沿用错误的符号只会更贵地失败一次。

### 4. 体型基因——预测被反向证伪，机制已诊断

**已做**：`size` 基因（`trait_dim` 2→3），只接水经济——代谢/耗水按体型 0.75 次方
（Kleiber）、`water_max` 按 1.0 次方（容积）。

**实测（§12.1）：预测是 [1.1, 1.4]，实测 `mean_size` = 0.725，六个种子无一在
1.0 以上。方向完全相反。** 体型冻结臂（`size_mutation_sigma=0`）实测 0.765，
说明**纯粹作用在 founder 标准变异上的选择**就能把均值拉下来，一个突变都不需要。

**根因已定量确认**：新生水量 = `invest_frac × 亲代水量` ≈ **2.28**，而
`min(..., water_max × 子代体型)` 的夹紧**从来不生效**（`size_min=0.4` 时容量 4.0
仍大于 2.28）。所以**出生水量与体型无关，而耗水按 `size^0.75` 增长**：幼体寿命从
size=0.4 的 227 步单调降到 size=1.6 的 80 步。`size^1.0` 的储水优势一次都没兑现过
——兑现它需要先活着走到水边把水箱装满，而那正是 83% 的个体做不到的事。

**"耐渴时间~体型^0.25"的推导没错，错在适用条件**：它假设水箱是满的。

**要让体型成为真正的权衡，必须让新生供给随子代自己的体型等比例放大**（现实中的
妊娠就是如此：更大的幼崽让母体付出成比例更多，出生时也成比例更满）。但这会改变
亲代投资的整个经济学（`invest_frac` 的含义随之改变）——**独立决策，不要顺手改。**

### 5. 果实层：三选一，尚未决定

`fruit_max=0` 六种子配对消融（`d342e34`）已判：果实层对空间行为完全没有作用
（`inland_frac` 差值 0.0006，CI ±0.018），唯一稳定效应是把 agent 拉进森林且这是
有害的（种群变少、平均年龄变短）。根因不是 logistic 陷阱，是量级——即使停在
logistic 峰值，果实全图供给也只有植物层的 0.35%。

三条路，代价都写清楚了（`docs/biology.md` §9.4）：
- (a) 放大到能被测试的量级（~50-60倍），风险：森林近水，可能加重 `inland_frac`
  倒退
- (b) 删掉，还回 2 个记忆槽 + 8 个输入，`in_dim` 59→51，**作废种群**，应与
  优先级 1/3/4 打包
- (c) 留着不动，明确承认是装饰

**这个决定应该和优先级 1（同类可见性）一起做**，因为两者都要动 `in_dim`，属于
"打包同一次种群作废"的候选。

### 6. mutation_sigma 变为可遗传基因（演化能力的演化，性价比高但容易被忽略）

对应用户原始诉求"生物进化的能力"。照抄 `invest_frac` 模式，`trait_dim` 加一维，
让每个谱系的突变率自身可演化。只动 `trait_dim`，比新增大脑输入便宜。

**测什么**：谱系间 `mutation_sigma` 基因值的方差是否随环境稳定性/波动性系统性
分化（经典预测：环境越稳定，最优突变率越低）。

**证伪**：与突变率固定为初始均值的对照臂比较长期适应度。

## 关键结论——不要重新调研或重新论证

- **死因分解**（六种子×20000步，~130万次死亡）：渴死 83.32%（平均死亡年龄 52.5
  步）、被捕食 9.95%（170.7步）、饿死 6.65%（353.0步）、老死 0.10%。平均寿命
  ~92步。83% 的死亡集中在一个原因一个年龄段，删失了其他一切性状的选择差——任何
  "某性状会被选择"的主张都要先过这一关（§8）。
- **幼体缺水瓶颈的数字全部对得上账**：新生水量 2.13，净耗 0.038/步→理论寿命
  56步（实测52.5步）；离水距离中位34.7 > 水感知半径30；记忆槽出生即空。是"有腿、
  有时间、没有方向"，不是速率问题（§8.3）。
- **果实层**：全图供给只有植物层的0.35%，六种子消融确认对空间行为无效果（§9）。
- **"两个物种"是焊死的，不是演化的**——四层机制列在优先级2（§10.1）。
- **气候稳定前提下，现实天花板是 Bedau Class 2（持续但有界），不是 Class 3**。
  没有一个被 Bedau 团队实测过的人工系统达到过 Class 3（§10.2）。
- **工具使用在"不显式指定人类属性"约束下不可达**——核实的每一篇文献（XLand、
  hide-and-seek）里，新颖性都来自设计者预置的物体/动作集合，没有案例展示 agent
  发明设计者没预留的交互方式（§10.3）。
- **可变神经网络拓扑在定形张量里已有发表方案**：TensorNEAT（GECCO 2024），NaN-pad
  到最大节点/连接数，JAX 下可 jit，实测最高544×加速（§10.5）。
- **交流不一定需要专用信道**（Quinn 2001：机器人复用移动动作演化出信号），但这是
  "交流"不是"工具使用"的证据，不影响工具使用不可达的判决（§10.5）。
- **体型基因治标不治本**：耐渴时间仅按体型~0.25次方增长，天花板低，不该当主力方案
  （体型报告）。
- **记忆不能遗传**（Weismann barrier）——已经决定，不要重新考虑。消融是欠功效
  （n=6，p=0.175，25%功效），不是零结果，但等价性检验证明效应<0.05（TOST
  p=0.032）（§5）。

## 不做的事

- 不要许诺"完全开放式演化"（Bedau Class 3）——文献没有任何先例支持这在静态环境
  +当前架构下可达。
- 不要做字面意义的"工具使用"（捡起/使用物体）——需要先塞入一个物体类别，直接
  违反"不显式指定人类属性"的约束。
- 不要把 ANNECS 当作本项目的指标——该指标专属"环境生成"路线（POET系），本项目
  地形静态，不适用。想要开放式演化的度量应该用 Bedau 式演化活动统计（按"表现型
  编码本"分桶计数），见 OEE 报告 Stage 6（未写入本文档，需要时翻对话记录或重新
  设计）。
- 不要重新加记忆遗传。
- 不要重新论证果实层的"logistic再生陷阱"——根因是量级不是陷阱，已经测过。
- 不要把优先级2的四层机制一次性全拆——按§10.5的两步法做。

## 统计纪律（贯穿所有实验）

6 配对种子起，或 5 个/组不配对；报告每个种子的数字，不只报均值；用 Mann-Whitney
或配对 Wilcoxon + bootstrap 区间；不做 Bonferroni 校正，报告每一个算出来的 p 值；
涉及空间指标的新结论需要跨地形种子（`ridge_wavenumber`/`ridge_amplitude`/
`ridge_base_y` 等），因为 `terrain.build` 不吃 RNG，同一份 `cfg` 下所有 founder
种子共享同一张地图——只跨 founder 种子的结论只对"这张地图"成立。

`scripts/run_headless.py --set FIELD=VALUE`（可重复传参）可以直接跑消融对照，
不用改 `config.py`，覆盖项会写进 `--json` 输出行。
