## 硬约束（每次动手前确认）

- **跑任何 JAX 进程都加 `XLA_PYTHON_CLIENT_PREALLOCATE=false`**。默认预分配 75%
  显存会让第二个进程假 OOM，实测真实峰值只有 918 MiB，加了就能十几个并行。
- **统计地板**：6 配对种子起，或 5 个/组不配对。报告每个种子的数字，不只报均值。
  Mann-Whitney 或配对 Wilcoxon + bootstrap 区间。**不做 Bonferroni，报告每一个
  算出来的 p 值。** 注意 n=6 配对符号秩能达到的最小双侧 p 就是 0.031——**0.031
  是地板，不是强证据。**
- **空间性结论需要跨地形种子**（`ridge_wavenumber`/`ridge_amplitude`/
  `ridge_base_y`），因为 `terrain.build` 不吃 RNG，所有 founder 种子共享同一张
  地图。只跨 founder 种子的空间结论只对"这张地图"成立，是伪重复。
- **消融用 `scripts/run_headless.py --set FIELD=VALUE`**（可重复传参），不要改
  `config.py`——那会让两个臂落在两棵不同的工作树上。覆盖项会写进 `--json` 行。
- **调研结论必须落进 `docs/`**，否则视为没做过。见 `CLAUDE.md` 同名一节。

### 未做——两条主线，按最新三份调研更新

**这三份调研（`rebalance` / `landscape_of_fear` / `trait_roadmap`）在同一个点会合：
幼体渴死瓶颈（81.9% 死亡）既删失了形态演化的选择差、又让捕食者靠河、又让"降种群"
的思路穿不过去。所以下面两条主线其实指向同一把钥匙。**

**主线 A —— 捕食者离水**（用户反复提的问题一）
- **不要走"减慢植物生长"这条路**——`rebalance.md` 三种子实测判决：它只碰资源总量、
  碰不到离水行为，温和减速让捕食者反而更贴水，激进减速直接灭绝捕食者。三目标同时
  成立的区间**不存在**。
- **正路是恐惧地景**（`landscape_of_fear.md`）。头号方案：恐惧场折入现有 `pred`
  通道，**零 `in_dim` 改动、不作废种群**，复用踩踏场的散射累加原语，且论证了不会
  重蹈踩踏"没有固定端点"的覆辙（它只需给已经集中的模式打时间平滑）。已标一个风险：
  公共信息可能延迟新生儿找水，需在实现时盯住 `death_thirst`。

**主线 B —— 性状/形态演化**（用户反复提的问题二）
- **`vision_radius` 可遗传是头号目标**（`trait_roadmap.md`）——它同时是 A 类性状
  （收益即时、不被幼体死亡删失）**又是**幼体渴死瓶颈的缓解路径（补 30 vs 34.7 的
  感知缺口）。对这个性状，"先修瓶颈"和"用性状演化修瓶颈"是同一件事。
- **`mutation_sigma` 可遗传**——最直接对应"生物进化的能力"，`trait_roadmap.md`
  给了逐行实现规格和区分"平凡下降 vs 真可演化性"的判据。
- **代价货币原则**（`trait_roadmap.md` 的核心洞见）：新性状代价记在**水账**会栽进
  幼体渴死陷阱（体型基因就是），记在**能账**才安全。任何新性状先过这一关。
- B 类性状（体型重设计、幼体期门控）**必须等瓶颈解除**，现在别碰。

**打包提示**：主线 A 头号方案不改 `in_dim`；主线 B 的 `vision_radius`/`mutation_sigma`
只动 `trait_dim`（便宜）。若要做会改 `in_dim` 的（危险记忆槽 A2、果实层删除、踩踏
"家"槽），应打包进同一次种群作废摊薄代价。

**零散**（与两条主线正交，随时可做）：
- **补上 peer 通道缺的那一环**：幼体到最近成体的平均距离 vs 随机置换基线。成本很低
  （run 结束对活体状态算一次），是"活得久一点"和"学会了跟随"之间唯一缺的证据。
- **踩踏全地形版**：`trample_path_gain` 接到 `move_cost`，排除"只在林冠下生效"。
- **果实层三选一**（放大/删除/承认装饰）。删除改 `in_dim`，打包做。

## 不做的事

- **不要许诺"完全开放式演化"（Bedau Class 3）**——没有任何已发表系统在静态环境
  下达到过。现实天花板是 Class 2。
- **不要做字面意义的工具使用**（捡起/使用物体）——需要先塞入一个物体类别，直接
  违反"不显式指定人类属性"的约束。
- **不要把 ANNECS 当作本项目的指标**——它专属"环境生成"路线（POET 系），本项目
  地形静态，不适用。
- **不要重新加记忆遗传。** Weismann barrier + 消融 + Rogers' paradox 三条独立
  理由同向。
- **不要重新论证果实层的"logistic 再生陷阱"**——根因是量级不是陷阱。
- **不要再调踩踏参数**——两个符号都测过了。
- **在做生态学重调之前不要再拆反混合装置**——单峰 founder 会直接饿死，测不到
  任何东西。
- **不要用"减慢植物生长/降资源总量"来赶捕食者离水**——三种子实测证伪（`rebalance.md`）：
  只碰资源总量、碰不到离水行为，还会灭捕食者。离水是资源的**空间结构**问题，不是
  总量问题。
- **不要单独下调 `carn_cost` 来"让捕食者活得起"**——安全但会加重"捕食者占比已经
  偏高一个数量级"的问题（`carnivore_riparian.md`），不是免费旋钮。
