# 约定背后的论证

`CLAUDE.md` 只放**指令**：做什么、不做什么、用哪条命令。这份文档放**为什么**——
那些指令各自是被什么代价买来的。

拆开的理由不是嫌 `CLAUDE.md` 长，而是它长到会伤害自己：一份塞满论证的指令文件
里，真正的硬约束会被淹没在解释中间。判据是官方那句——逐行问"删掉它会让 Claude
犯错吗"，会的留在 `CLAUDE.md`，不会但值得记住的搬到这里。

还有一层：下面几条约定现在有了机械执行者（`.claude/hooks/` 与
`scripts/check.py`），指令就不必再靠篇幅去说服人遵守了。**一条被 hook 挡住的规则
不需要写三遍。**

---

## 1. 为什么每个 python 进程都要 `XLA_PYTHON_CLIENT_PREALLOCATE=false`

`CUDA_ERROR_OUT_OF_MEMORY` 在这个项目里几乎总是预分配的假象，不是真的显存不够。
JAX 启动时会一次性抓走显卡的 75%（24 GB 的卡上是 18.5 GB），所以**第二个**启动的
进程会失败——而一次满规模运行的真实峰值只有 **918 MiB**（实测，`n_max=16384`）。

加上这个变量之后十几个进程可以共存，多种子 sweep 就是这么并行跑的。不加，你是在
毫无理由地把作业串行化；而且一个正在跑的 `run_live.py` 会看起来像"占住了整张卡"
（它没有，它占住的是预分配）。

**执行者**：`.claude/hooks/require-preallocate.sh`。它在 `PreToolUse(Bash)` 上拦截
命令位置上的 python 调用，缺变量就 deny 并把改好的命令回给你。之所以是 deny 而不
是 ask：修法是机械的，且不存在"这次就不加"的正当情形。

## 2. 为什么 commit 按"变更原因"拆而不是按特性拆

一个有用的判据：**如果 commit message 里需要出现"顺便"或"我在那儿的时候"，它就该
是两个 commit。**

代价是实测过的。曾有一个 commit 把新的资源层、一次线协议版本号提升、以及一处
shader 改动打包进 14 个文件——那应该是三个，因为后来的读者想查"协议是什么时候变
的"，得先读一段关于果实生态学的说明才能找到。

commit message 用中文写（标题与正文都是）。两样东西保持原样：`Co-Authored-By:`
trailer（它被工具解析，不是给人读的散文），以及从代码里引用的标识符
（`carn_cost`、`world_step`、`UNTR`、文件路径）——那些是名字，不是描述。

正文的标准和用英文写时一样：说清楚**为什么**改、量到了什么，而不只是动了哪里。

```
为地形系统增加山脉、河流与森林

将世界扩大到 512²，让"待在哪里"本身成为值得演化的决策……

Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

## 3. 为什么调研结论必须落进 `docs/`

只存在于对话里的报告会在下一次 context 压缩时消失。**这件事已经发生过一次**：一份
完整的三维化可行性研究做完了、口头转述了、没有人据此行动，然后就没了，几周后同一
个问题只好从零再问一遍。

所以规则是：任何产出为结论而非代码的调查，**必须**写进 `docs/` 下的文件并提交，才
算做完。

- 一个主题一个文件 `docs/<topic>.md`。长没关系。这些文档被允许——而且被鼓励——写
  得像文献综述：完整的散文、真实的引用（作者、年份、发表处）、表格、推导、死胡同。
  它们不是 prompt，不该被压缩成要点式指令。
- **每条断言都标注它是怎么被确立的。** `docs/biology.md` 用 `[现实]`（已发表事
  实）、`[本世界实测]`、`[对应]`（落在代码哪里）、`[提案，非结论]`。沿用这套词
  汇。来源核实不了就直说，不要把它洗成自信的散文。
- 在 `docs/TODO.md` 加一行指针，下一个 session 才找得到。
- 负面与不确定的结果是重点，不是难堪——`docs/biology.md` 的 §9、§11、§12、§13 大
  部分是失败，而它们是整个文件里最吃重的部分，因为它们阻止同一个想法被反复重试。

## 4. 记忆不可遗传：那段被修正过的论证

新生儿的记忆槽是**空的**。基因跨代，记忆在一生之内获得、随持有者死去——出生时复制
父代的槽是拉马克式的。已在 `cbe434d` 移除，不要重新引入。

**这段文字的早期版本说该复制"也被实测为没有作用"。那是一次 overclaim，值得作为警
示留下来。** 数据（n=6 配对，`inland_frac` 均值 +0.020，配对 SD 0.031）给出
p=0.175，95% 区间 [−0.013, +0.053]，**功效 25%**——即便真实效应恰好就是观测到的
+0.020，得到零结果也仍是最可能的结局。

数据真正支持的，是等价检验给出的那个更窄的结论：任何效应都**小于 0.05**
（TOST p=0.032）。移除这个机制依然是对的，因为拉马克式论证本身就足够站得住，举证
责任在机制那一边——但诚实的描述是"功效不足，上界 0.05"，不是"没有作用"。

跨代传递知识的正当路线是社会学习：幼体跟随成体并自己饮水。`memory.write` 已经支持
这条路，因为它只问你是否站在水边，不问你怎么到的。

空槽起步是真实的劣势：实测头 50 步的风险率是老年期的 **17.7 倍**，全部死亡的 63%
发生在那里。见 `docs/biology.md` §6。

## 5. 统计地板是怎么算出来的

**六个配对种子，或每臂五个不配对——三个在地板以下。** 这条取代了此前"≥3 种子"的
规则，那条规则在算术上就不可能产出结果：3-vs-3 Mann-Whitney **能达到的最小**双侧
p 是 0.10，3 对符号秩检验是 0.25，无论数据长什么样。每臂 4 个不配对、6 个配对，是
检验第一次有可能触及 0.05 的样本量。

实测的种子间 SD 是 `inland_frac` ±0.027、`carnivore_frac` ±0.012，所以要在 80% 功
效下检出 `inland_frac` 0.02 的移动需要 **约 21 个配对种子**——照此编预算，或者直说
这次运行功效不足。

n=6 配对符号秩能达到的最小双侧 p 就是 **0.031**，所以报告里的 0.031 是在地板上，
不是强证据。

不做 Bonferroni 校正，把你算过的每一个 p 值都报出来。报告每个种子的数字，不只报均
值——`--json` 已经把它们吐出来了。

## 6. 空间性结论为什么需要跨地形种子

`terrain.build(cfg)` 完全不用 RNG，所以**每个种子跑的是同一张地图**。任何空间性断
言（`inland_frac`、`water_bound_frac`、那两个距离指标）因此只推广到*这一套河
系*，不是推广到一般的河流——那是伪重复。

诚实的修法是让 `ridge_wavenumber` / `ridge_amplitude` / `ridge_base_y` 以及河源随
一个地形种子变化，然后把地形与创始者当作交叉因子处理。注意刚性平移不够：世界是环
面，把地图整体挪一下什么都不改变。

## 7. 空间指标在有零假设之前没有意义

`inland_frac = 0.30` 在你知道随机放置会给出多少之前，不叫"低"。仅从地形算出来：
在所有格子上均匀 → 0.556，在可居住格子上均匀 → 0.675，按承载力加权 → 0.650。

所以种群坐在比随机**低 0.35** 的位置上，这才是真正的发现；而一个 +0.020 的效应是
那个差距的 5.8%——这才是诚实的量级表述。

## 8. 生态参数是实测调出来的，不是随手填的

`config.py` 里带着很长的注释，记录试过什么、值为什么在那里（`plant_max`、
`regrow_baseline`、`attack_range`、`carn_cost`、`n_init`）。好几个看起来合理的改动
被测过并否掉了，因为它们在 20k+ 步里把捕食者逼到灭绝。

**永远不要在单次运行上调生态。** 捕食者存活是一个近阈值的随机过程，run-to-run 方差
比大多数参数效应都大。一个在单个种子上看起来明显最好的配置，在测过的全部四个种子
上都拿到 0% 捕食者；而一个看起来明显最差的配置平均有 2%+——整个第一轮结论都是噪声。

`n_init` 有一个在两个方向上都很窄的甜点。播种太接近 ~700 的平衡态，捕食者创始池小
到扛不住自身的随机性（早期版本在 `n_init=280` 下到第 800 步就死光）；播得远高于
它，初始的暴跌本身就会杀死它们（`n_init=4800` 时 7 倍的崩塌在每个测过的种子上都抹
掉了捕食者）。大约 3 倍于预期平衡态是可以的。

`carn_cost` 在平坦的 256² 世界里是 0.15；带地形的世界要向捕食者收取爬坡费用、并在
植被中减慢它们，0.15 下它们在每个测过的种子上都灭绝了。0.10 恢复了约 15% 的捕食者
比例（三个种子上实测）。

## 9. 为什么 GPU 上的确定性不是逐位的

按格子的 scatter-add 是原子操作且会重排，所以 `test_determinism` 断言的是相同的
生死结构加上短程上的数值容差，而不是逐位相等。要真正的位确定性，用
`XLA_FLAGS=--xla_gpu_deterministic_ops=true` 跑。

这也是 `scripts/check.py` 的 golden 带宽存在的原因。不过实测：在 smoke 配置
（2048 agents、200 步）上连跑五次，全部十个指标漂移 **0.000%**——重排在这个规模下
没有显现。所以那些带宽不是由噪声决定的，是由**粒度**决定的：种群 832 时一个 agent
是 0.12%，而 `carnivore_frac` 上一个 agent 是 5.6%，后者因此拿到最宽的带宽，尽管
它在绝对值上最稳定。

带宽如果开始 flake，先重新测量再放宽——**为了让失败消失而放宽的带宽等于删掉了这项
检查。**

## 10. 眼睛能看到的东西必须用眼睛验

一个让植物场渲染成一整块饱和色板的 shader bug 长期没被发现，因为它只被"推理"过，
从来没有被看过。截图（headless chromium 加
`--use-gl=angle --use-angle=swiftshader --enable-unsafe-swiftshader`）能抓到 code
review 抓不到的东西。

线协议的偏移量同理，但那一类现在有机械检查了：`scripts/check.py` 会把
`server/protocol.py` 的 struct 格式串逐字段展开，和 `web/main.js` 里每一个
`getFloat32` 偏移量对齐比较，并核对三个文件里重复的物种颜色。**一个错误的偏移量产
出的是看起来合理的错误数字，不是异常**——这正是它该由检查器而不是由 reviewer 的记
忆来守的原因。
