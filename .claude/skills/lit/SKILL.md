# 文献检索并入 `docs/`

对象：$ARGUMENTS

本 skill 在隔离子上下文中运行：检索过程产生的大量原文不会污染主 context，
只有最终写进了哪个文档、哪几篇、为什么不入，会回到主线程。

> 为什么显式写 `background: false`：`context: fork` 的 skill 默认后台运行。本 skill 的产出
> 是给用户当场看的判断依据（入没入档、哪几篇、哪几篇核不到），后台跑会让它变成事后通知。
> 隔离 context 要保留，后台不要。

> **三个落点，方向是单向的：`notes/papers/` → `lit/literature-log.md` → `docs/<topic>.md`。**
> `notes/papers/<citekey>.md` 是逐篇原始笔记，`lit/literature-log.md` 是可扫的台账，
> **`docs/<topic>.md` 是唯一的权威成品**（`docs/biology.md` 就是一份完整的、带真实引用的
> 综述，这个体例不变）。一篇论文的价值在于它支持或推翻了本项目的哪个设计判断，那句话
> 必须和设计写在同一处——所以**绝不允许出现「结论只存在于 `notes/` 而没进 `docs/`」**，
> 那违反 `CLAUDE.md` 的「Research lands in `docs/`, or it did not happen」。
> 前两者存在的理由是本项目缺一条可查台账：`docs/biology.md` 里的引用没法一眼看出
> 「读过没有、当初查的什么 query、结论进了哪几份文档」，两次引用事故（大象幼崽死亡率
> 的统计层级错位、虎鲸「约 8 倍」无文献支持）都发生在这个盲区里。详见
> `notes/papers/README.md` 与 `lit/literature-log.md` 的开头。

## 第 0 步 先查有没有已经写过

`docs/` 下已经有大量带引用的正文。**重复入档比不入档更糟**——同一篇论文在两处措辞不同，
以后没人知道信哪个。台账先查，它是可扫的那一份；`docs/` 再查，它是权威的那一份：

```bash
grep -n "<第一作者姓>\|<论文标题关键词>\|<arXiv ID>\|<DOI>" lit/literature-log.md
ls notes/papers/ | grep -i "<第一作者姓>"
grep -rn "<第一作者姓>\|<论文标题关键词>\|<arXiv ID>" docs/
```

命中就直接报告「已在 `docs/xxx.md` 的 <小节> 引用过（台账 `lit/literature-log.md`，笔记
`notes/papers/<citekey>.md`）」并停止；若发现那处引用与本次检索到的原文对不上（年份、期刊、
数字、**统计层级**），那就是本次真正的产出：**报告出入，不要另写一份**。

台账里有行、但「已并入 docs/」写着「未并入」的，说明上次读了没用上——这次要么把它并进去，
要么在台账里写清为什么不并。**检索本身也要查重**：`lit/literature-log.md` 的「检索记录」
一节记了历史 query，包括没有命中的那些，别把同一条 query 再跑一遍。

顺便确认目标文档和小节：默认落点是 `docs/biology.md`（真实世界生态学与行为学的基础），
机制性/工程性的相关工作按主题落到对应文档（3D 走 `docs/three_d.md`，开放式演化走
`docs/trait_evolution.md`，等等）。**主题在 `docs/` 里没有归宿时，先问用户要不要新建
`docs/<topic>.md`，不要自作主张开新文件。**

## 第 1 步 检索

按顺序选工具，能用前面的就别用后面的：

1. **已配置的 MCP**（`zotero` / `paper-search` / `semantic-scholar` 之类）——先看本会话
   有哪些可用。
2. **WebSearch / WebFetch** —— 搜到之后**必须实际打开页面确认**，不能只凭搜索结果摘要。

主题较宽（要一次覆盖 5 篇以上）时派 subagent 并行检索，每个负责一个子问题，回报
「标题 + 年份 + 期刊/会议 + DOI 或 arXiv ID + 三行摘要 + 用到的具体数字及其表/图编号」。

传入的是 URL / DOI / arXiv ID 时跳过检索，直接取那一篇。

## 第 2 步 核实（本 skill 的硬核心）

对每一篇，逐项核对：标题、全部作者、年份、期刊/会议、卷期页码、DOI 或 arXiv ID、
以及**你打算引用的每一个数字**。数字要能指到原文的表/图/正文位置。

**核不到就明说核不到。** `CLAUDE.md` 的原话是：Say plainly when a source could not be
verified rather than laundering it into confident prose。写成
「检索了 <关键词>，在 <工具> 里没有找到匹配的论文」或「找到条目但拿不到全文，
未核实卷期与数字」，**不要给一个"大概是这篇"的猜测**。

## 第 3 步 写逐篇笔记 + 记台账

**先写笔记，再并入 `docs/`。** 顺序不能反：从原文直接跳到散文只有一跳，中间没有落脚点，
搬运走样时没有任何东西能挡住——本项目两次引用事故就是这么发生的。

每篇入选的论文写一份 `notes/papers/<citekey>.md`。citekey 是
`<第一作者姓小写><年份><标题第一个实词小写>`（`parker2021orphan`）。frontmatter 字段与
正文六节（核心问题 / 方法 / 关键结论 / 与本项目的关系 / 可复用的点 / 存疑）**照
`notes/papers/README.md` 的模板抄，不要自创字段**。两处容易漏的：

- 「方法」里必须写清**统计层级**（族群平均还是个体分组、相对风险还是绝对比例、样本量）。
  上面两次事故都是在这一格没写清才发生的。
- 「关键结论」里每个数字都要注明出自哪张表 / 哪张图 / 正文哪一节。指不到就不写。

然后在 `lit/literature-log.md` 加一行，列顺序固定：
`citekey | 标题 | 年份 | 状态 | 与本项目的关系 | 笔记路径 | 已并入 docs/`。
本次检索本身（包括**没有命中**的 query）追加进同文件的「检索记录」表。

**判定为不相关的论文也要记台账**（状态 `已弃`，「已并入」写「未并入：不相关」）——
「查过、判定不相关」是对下一次检索最有用的一条信息，不写就等于没查过。
不相关的不写 `notes/papers/` 笔记，一行台账即可。

## 第 4 步 并入 `docs/<topic>.md`

写进目标文档的**相关小节**（不是文末堆一个"参考文献"列表——本项目的文档是散文体综述，
引用嵌在论证里）。格式对齐 `docs/biology.md` 已有的写法：

````markdown
**供给（provisioning）——文献里最干净的数字。** Parker et al. 2021, *Current Biology*
31:4156–4162，Samburu 大象 19 年数据：0–2 岁的依赖期孤儿存活概率**恰好为 0**，
对比非孤儿 **0.951 [0.917, 0.968]**。
````

规矩：

- **每段按四标签标注**：`[现实]` 已发表事实（本 skill 的主要产出）、`[本世界实测]` 本项目
  测出的数字（**本 skill 不产生这类，不要给文献结论贴这个标签**）、`[对应]` 它落在代码
  哪个参数/机制上、或明确写"尚未实现"、`[提案，非结论]` 我据此提出的设想，**不得当结论引用**。
- **区分"论文说的"和"我的判断"**。后者一律 `[提案，非结论]`。
- 人名、文献标题、术语（piosphere、Hamilton's rule）、标识符保持原文，不翻译。
- 跨文档引用写**文件名加小节标题**（"见 `docs/mortality.md` 的死因分解"），不要写裸的 `§N`。
- **反面证据照收**。找到的文献如果**推翻**了本项目某条设计假设，那正是最有价值的一次入档；
  同时在该文档里写清它推翻的是哪一条。
- 与本项目没有实质关联的论文**不要入档**——直接在回报里说"检索到但不相关，未写入"，
  台账里那一行照记。
- **`docs/` 里的段落是自足的**：读的人不该为了看懂一条结论去翻 `notes/`。笔记是出处，
  不是正文的一部分，所以不要在 `docs/` 里写「详见 `notes/papers/xxx.md`」来代替把话说完。

并入之后**回填台账**：把该行的「已并入 docs/」写到小节级（例：`docs/biology.md` 的
「亲代照料」），状态改成 `已并入`。这一列是唯一能一眼看出一篇论文有没有真正落地的地方，
漏填等于这次入档没发生。

新建了 `docs/<topic>.md` 的话，**在 `docs/TODO.md` 的文档地图表加一行指针**
（文档 / 内容 / 什么时候读它），否则下一个 session 找不到它。

## 硬约束

1. **引用必须来自本次真实检索结果**，并且带 DOI 或 arXiv ID（或可核实的期刊卷期页）。
   没有标识符的条目不入档。
2. **严禁凭记忆生成 BibTeX、标题、作者、年份或页码。** 训练数据里的文献信息足够似是而非到
   骗过 review，这是最不能容忍的一类错误。
3. **数字必须能指到原文的表/图编号**，写进文档时注明出处。记不清就不写。
4. **不得给文献结论贴 `[本世界实测]`。** 那个标签只属于本项目跑出来的数字——
   它证明的是"我们的模型做到了 X"，不能用来论证"自然界如此"，反过来也一样。
5. **不改已有段落的结论**。发现旧引用有误，就在旁边补一段写明哪里对不上、依据是什么，
   由用户决定怎么改，不要静默重写。
6. **笔记不能替代入档。** 只写了 `notes/papers/` 而没有任何东西并进 `docs/`，只在「这篇
   确实不相关」时成立，而且要在台账里写明理由。否则那条结论等于没做过。
7. 不动源代码。本 skill 只写 `docs/`、`notes/papers/` 和 `lit/literature-log.md`。

## 回到主线程时输出

只回报五样，别把检索原文带回去：

1. 写进了哪个文件的哪个小节（`docs/xxx.md` 的 <小节标题>），新增几段
2. 新增/更新了哪几份 `notes/papers/<citekey>.md`，台账加了几行
3. 每篇一行：`作者 年份` + DOI/arXiv ID + 它支持/推翻了本项目的哪个判断
4. 明确列出「检索了但没入档」的和原因（不相关 / 拿不到全文 / 已存在），以及它们在台账里
   记成了什么状态
5. **核不到的条目单独列一节**，原话说明核不到什么
