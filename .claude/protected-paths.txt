# ============================================================
# 受保护路径清单  —— 由 .claude/hooks/protect_paths.py 读取
# ============================================================
# 列在这里的路径，Claude 的 Edit / Write 会被阻断（exit 2）。
#
# ------------------------------------------------------------
# 两类规则（务必看懂）
# ------------------------------------------------------------
#   普通规则（不带前缀）—— **仅当目标文件已存在时**才阻断，新建放行。
#   `!` 前缀规则       —— **无条件阻断**，不管文件存不存在。
#
#   这里的 `!` **不是 gitignore 的取反语义**。它就是"更严格"的意思。
#
# 格式：每行一个 glob，`#` 开头是注释，空行忽略。
# 规则同时用「相对项目根的路径」和「纯文件名」匹配，`*` 会跨越 `/`。
# 临时放开某条：注释掉 → 让 Claude 改 → 立刻恢复注释。
# 本文件不存在时，hook 使用与下面等价的内置默认值；
# 若本文件存在但全是注释/空行，视为「不保护任何路径」。
# ------------------------------------------------------------

# --- golden band：本项目唯一真正怕被"顺手改掉"的文件 ---
# CLAUDE.md：「Never widen the bands to make a failure go away -- a band widened
# for that reason is a check deleted.」
# 手改 golden.json 正是"放宽 band 让失败消失"的操作路径，而且改完不留痕迹。
# 合法路径是 `scripts/check.py --bless`（走 Bash，本 hook 不拦）+ 在 commit
# message 里说明为什么这次改动**应该**挪动这些数字。
# 无条件：新建一份假的 golden.json 和改一份真的一样有害。
!scripts/golden.json

# --- 密钥：既不该被读，更不该被写，也不该被凭空造出来 ---
!.env
!.env.*

# --- 锁文件：手改会破坏环境可复现性，应由包管理器自己生成 ---
!*.lock
!uv.lock

# --- 版本库内部：改了等于毁仓库 ---
!.git/**

# ------------------------------------------------------------
# 刻意**没有**保护的东西，以及理由
# ------------------------------------------------------------
# outputs/ checkpoints/ runs/
#     模板默认守这几个（科研场景里跑三天的结果没有 undo）。本项目不适用：
#     它们已经在 .gitignore 里，是可重跑的临时产物，不是版本库内的只读产出区。
#     守它们只会在正常写日志时误伤。
#
# underworld/config.py         <- 考虑过，决定不加，理由如下
#     诱惑很大：CLAUDE.md 说这里的生态参数是实测调出来的，
#     「Several plausible-looking changes were tested and rejected」。
#     但它同时是**最活跃的开发文件**——水修复（543483a）、carn_cost、
#     恐惧场参数，每一次落地都要改它。把它挂上无条件阻断，等于把项目的主线
#     工作变成每次都要人工放行，几天之内这个文件就会被注释掉，连带 golden
#     那条也一起失效（护栏一旦被当成噪音，是整个被关掉的）。
#     而且它已经有两道更合适的防线：改坏了 golden band 会在 tier 2 抓到，
#     消融按 CLAUDE.md 的要求走 `run_headless.py --set FIELD=VALUE` 根本不碰它。
#     护栏应该守「改了没痕迹」的东西，config.py 恰恰是改了一定有痕迹的那种。
#     真要打开就把下面这行的 `#` 去掉（普通规则，仅拦修改已有文件）：
# underworld/config.py
#
# docs/**
#     CLAUDE.md 明确要求调研结论必须落进 docs/，拦它等于跟项目规矩对着干。
